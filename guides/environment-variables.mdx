---
title: Environment variables
description: Configure the SDK with server-side environment variables.
---

## Required

```env
MARGOVIA_API_KEY=mg_live_xxx
```

The API key authenticates events sent to Margovia. Keep it server-side.

If `MARGOVIA_API_KEY` is missing, the SDK becomes a no-op. This is useful for local development but means no runs or cost events will appear in Margovia.

## Optional

```env
MARGOVIA_BASE_URL=https://api.margovia.com
```

The SDK defaults to `https://api.margovia.com`.

Set `MARGOVIA_BASE_URL` only for local development, staging, or self-hosted API testing:

```env
MARGOVIA_BASE_URL=http://localhost:4010
```

```env
MARGOVIA_DEBUG=1
```

Enables debug logging for skipped or failed tracking operations.

Use this when you are verifying an integration. Leave it off in normal production logs unless you are troubleshooting.

## Provider keys

Provider API keys are not used by Margovia directly. They are used by your app and provider SDKs:

```env
OPENAI_API_KEY=sk_xxx
ANTHROPIC_API_KEY=sk-ant-xxx
```
